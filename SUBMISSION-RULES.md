# Submission Rules (TPP Py4E Taglish Track)

## 1) One PR per module
- Submit using **Pull Requests**
- Keep **1 PR per module** until it passes

## 2) Naming conventions
- Branch: `module-XX-<short>`
- PR Title: `Module XX - <Title>`

## 3) Required PR checklist
Your PR must include:
- ✅ Code that runs
- ✅ Evidence (screenshot or pasted output)
- ✅ Reflection answers (in PR description)
- ✅ Mastery gate coding check file(s) if required

## 4) Mastery gate (80% to unlock)
- Reflection: 20%
- Coding check: 80%
- Passing score: **80%**

If you score <80%:
- You will receive **Corrective Actions**
- You must update the **same PR** (no new PR) until you pass

## 5) Academic honesty + AI use disclosure
- You can discuss ideas, but your final code must be **your own**
- If you used help (AI, friends, docs), disclose in PR notes: “Help used: ___”

## 6) Safety: never commit secrets
Do NOT commit:
- API keys
- passwords
- OAuth client secrets
- personal data

If you accidentally pushed secrets:
- Immediately delete/revoke the secret
- Tell the instructor
